# 03 - 会话持久化（sessions.json + <session_id>.jsonl）

> 来源：[`claw0/sessions/`](../claw0/sessions/)（运行时生成；claw0 仓库里没有示例，需要跑一次才有）
>
> [[03 - Sessions]] 的 `SessionStore` 用**两层存储**：
> - **`sessions.json`** —— 会话索引（所有会话的元信息）
> - **`<session_id>.jsonl`** —— 单个会话的对话历史（每行一个事件）

## 目录结构

```
workspace/
└── sessions/
    ├── sessions.json              ← 索引（所有会话）
    ├── a3f7b9c2d1e4.jsonl         ← session a3f7b9c2d1e4 的对话历史
    ├── f8e1d4a6b2c9.jsonl         ← session f8e1d4a6b2c9 的对话历史
    └── ...
```

## sessions.json（索引文件）

记录**所有会话的元信息**——ID、标签、创建时间、最近活动时间、消息数。

```json
{
  "a3f7b9c2d1e4": {
    "label": "Telegram chat with @puin",
    "created_at": "2026-06-21T10:30:00+00:00",
    "last_active": "2026-06-23T14:22:11+00:00",
    "message_count": 47
  },
  "f8e1d4a6b2c9": {
    "label": "CLI session",
    "created_at": "2026-06-22T09:15:00+00:00",
    "last_active": "2026-06-22T11:42:30+00:00",
    "message_count": 12
  }
}
```

**字段说明**：
- key 是 `session_id`（`uuid.uuid4().hex[:12]` 生成的 12 位 hex）
- `label` 人类可读标签（可空）
- `message_count` 在 `append_transcript` 时 +1（s03_sessions.py L209）

**用途**：`/sessions` REPL 命令列出所有会话；session 切换时按 id 查找。

## `<session_id>.jsonl`（对话历史）

**JSONL 格式 = 每行一个 JSON 对象**——比 JSON 数组更适合追加写入（不需要读整个文件才能 append）。

```
{"type": "user", "content": "你好 Luna", "ts": 1718956200.123}
{"type": "assistant", "content": "你好！今天怎么样？", "ts": 1718956200.891}
{"type": "user", "content": "帮我看看 sessions 目录", "ts": 1718956210.456}
{"type": "assistant", "content": [{"type": "text", "text": "好的，让我看看"}, {"type": "tool_use", "id": "toolu_01abc", "name": "list_directory", "input": {"path": "sessions"}}], "ts": 1718956211.234}
{"type": "tool_use", "tool_use_id": "toolu_01abc", "name": "list_directory", "input": {"path": "sessions"}, "ts": 1718956211.235}
{"type": "tool_result", "tool_use_id": "toolu_01abc", "content": "sessions.json\na3f7b9c2d1e4.jsonl\nf8e1d4a6b2c9.jsonl", "ts": 1718956211.301}
{"type": "assistant", "content": "sessions 目录里有 2 个会话文件和 1 个索引", "ts": 1718956212.012}
{"type": "user", "content": "切换到 a3f7 那个", "ts": 1718956220.500}
```

### 5 种事件类型

| `type` | 字段 | 说明 |
|---|---|---|
| `user` | `content`（str） | 用户消息 |
| `assistant` | `content`（str 或 list[block]） | 模型响应；可能是纯文本或 content blocks（含 text + tool_use） |
| `tool_use` | `tool_use_id`, `name`, `input` | 工具调用（模型决定调用什么工具） |
| `tool_result` | `tool_use_id`, `content` | 工具结果（执行完返回给模型） |
| `system`（可选） | `content` | 系统事件（如 cron 投递的 message） |

### 为什么用 JSONL 而不是 JSON

- **JSON 数组**：要 append 必须读全文件 → parse → push → stringify → 写回。10MB 文件 append 一条要 100ms。
- **JSONL**：直接 `f.write(json.dumps(record) + "\n")`，O(1) 追加。10MB 文件 append 一条 < 1ms。

代价：**不能直接 `json.load()`**，要逐行 parse。但 s03 的 `_rebuild_history` 已经封装好：

```python
# s03_sessions.py 的 _rebuild_history

    for line in path.read_text(encoding="utf-8").splitlines():
        if not line.strip():
            continue
        record = json.loads(line)
        # 把 record 转回 API 格式的 message
        ...
    return messages
```

## 会话切换的语义

```
启动 →  default session 或上次的 session
       ↓
       load_session(id) → _rebuild_history → messages[]
       ↓
       agent_loop 用 messages[] 跟模型对话
       ↓
       save_turn/save_tool_result → append_transcript → 写到 <id>.jsonl
       ↓
       /switch <other_id> → 切换：load_session(other_id)
```

**关键**：messages[] 是**易失的内存状态**，jsonl 是**持久的磁盘真相**。切换 session 时丢掉内存 messages、从 jsonl 重建——保证一致性。

## 多通道共享同一 session 的场景

用户在 Telegram 和 CLI 都跟同一个 agent 对话：

```
Telegram 消息 → session_key = "telegram:<user_id>"
                     ↓
                sessions/a3f7b9c2d1e4.jsonl
                     ↓
CLI 消息    → session_key = "cli:default"
                     ↓
                sessions/f8e1d4a6b2c9.jsonl
```

**不同通道 = 不同 session**（默认行为）。如果想让"Telegram 和 CLI 共享对话历史"，[[04 - Channels]] 的 `build_session_key` 改成 `return "shared"` 即可。

## 改造成 PuinClaw 时

PuinClaw 的 goal-runner 也用类似的 JSONL 持久化（`messages.json`）。差别：

| claw0 session | PuinClaw goal-runner |
|---|---|
| `sessions/<id>.jsonl` | `goal_dir/messages.json` |
| JSONL 一行一条 | JSON 数组（整个 messages） |
| 多 session 切换 | 单 goal 单 messages |
| `save_turn` 追加 | runner 整体覆盖 |

claw0 的 JSONL 适合**长期多会话**的 IM bot，PuinClaw goal-runner 的 JSON 适合**单 goal 几百轮**的批处理任务。
